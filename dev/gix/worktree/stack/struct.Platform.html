pub fn excluded_kind(&self) -> Option<Kind>

See if a non-negative ignore-pattern matches and obtain the kind of exclude, or return None if the path isn’t excluded.

This is similar to is_excluded(), but provides details that are useful to decide what to do with the excluded item.
