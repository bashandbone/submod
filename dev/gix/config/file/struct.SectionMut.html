pub fn push<'b>( &mut self, value_name: ValueName<'event>, value: Option<&'b BStr>, ) -> &mut SectionMut<'_, 'event>

Adds an entry to the end of this section name value_name and value. If value is None, no equal sign will be written leaving just the key. This is useful for boolean values which are true if merely the key exists.

pub fn push_with_comment<'b, 'c>( &mut self, value_name: ValueName<'event>, value: Option<&'b BStr>, comment: impl Into<&'c BStr>, ) -> &mut SectionMut<'_, 'event>

Adds an entry to the end of this section name value_name and value. If value is None, no equal sign will be written leaving just the key. This is useful for boolean values which are true if merely the key exists. comment has to be the text to put right after the value and behind a # character. Note that newlines are silently transformed into spaces.

pub fn pop(&mut self) -> Option<(ValueName<'_>, Cow<'event, BStr>)>

Removes all events until a key value pair is removed. This will also remove the whitespace preceding the key value pair, if any is found.

pub fn set( &mut self, value_name: ValueName<'event>, value: &BStr, ) -> Option<Cow<'event, BStr>>

Sets the last key value pair if it exists, or adds the new value. Returns the previous value if it replaced a value, or None if it adds the value.

pub fn set_leading_whitespace( &mut self, whitespace: Option<Cow<'event, BStr>>, ) -> &mut SectionMut<'_, 'event>

Sets the exact whitespace to use before each newly created key-value pair, with only whitespace characters being permissible.

The default is 2 tabs. Set to None to disable adding whitespace before a key value.

pub fn leading_whitespace(&self) -> Option<&BStr>

pub fn separator_whitespace(&self) -> (Option<&BStr>, Option<&BStr>)

Returns the whitespace to be used before and after the = between the key and the value.

For example, k = v will have (Some(" "), Some(" ")), whereas k=\tv will have (None, Some("\t")).
